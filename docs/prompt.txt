KONSEP FINAL SISTEM
===================

Public website bebas diakses tanpa login.

User bisa melihat:

* Landing page
* Daftar paket wisata
* Detail paket
* Galeri
* Review
* Lokasi
* Informasi destinasi

Tetapi ketika user klik Booking/Pesan Sekarang, user wajib punya akun.

Flow final:

Landing Page
→ Detail Paket
→ Klik Booking Sekarang
→ Auth Check

* Jika belum login: redirect ke Login/Register
* Jika sudah login: masuk Booking Form
  → Checkout
  → Bayar via Midtrans
  → Payment paid via webhook
  → Booking confirmed
  → E-ticket dibuat otomatis
  → Pesanan Saya

PENTING:

* User tidak punya dashboard.
* User hanya punya Pesanan Saya.
* Admin tetap punya dashboard.
* Booking tidak lewat WhatsApp.
* Payment tidak upload bukti manual.
* Payment utama wajib via Midtrans Snap.
* Status paid/confirmed hanya boleh diupdate dari webhook Midtrans, bukan dari JavaScript frontend.

MIDTRANS PAYMENT
================

Gunakan Midtrans Snap.

Install package:
composer require midtrans/midtrans-php

Env:
MIDTRANS_SERVER_KEY=
MIDTRANS_CLIENT_KEY=
MIDTRANS_IS_PRODUCTION=false
MIDTRANS_IS_SANITIZED=true
MIDTRANS_IS_3DS=true

Config:
Buat config/midtrans.php

Isi:

* server_key
* client_key
* is_production
* is_sanitized
* is_3ds

Jangan hardcode key di controller atau Blade.
Server key tidak boleh muncul di frontend.
Client key boleh dipakai frontend untuk Snap JS.

Buat service:
app/Services/MidtransService.php

Methods:

* createSnapToken(Booking $booking): string
* handleNotification(array $payload): void
* validateSignature(array $payload): bool

createSnapToken:

* Setup config Midtrans.
* Ambil payment dari booking.
* Gunakan payment.order_id.
* gross_amount dari booking.total_price.
* customer_details dari user/contact.
* item_details dari package.
* Simpan snap_token ke payment.
* Return snap_token.

Webhook:
Route:
POST /payments/midtrans/notification

handleNotification:

* Validasi signature key.
* Cari payment by order_id.
* Simpan raw_response.
* Update transaction_status, payment_type, fraud_status.
* Jika settlement/capture success:

  * payment.status = paid
  * payment.paid_at = now()
  * booking.status = confirmed
  * generate ticket jika belum ada
* Jika pending:

  * payment.status = pending
  * booking.status tetap waiting_payment
* Jika expire:

  * payment.status = expired
  * booking.status = expired
  * ticket expired jika ada
* Jika cancel/deny/failure:

  * payment.status = failed/cancelled
  * booking.status = cancelled

Gunakan DB::transaction.

SNAP JS FRONTEND
================

Di checkout page:

* Load Snap JS:
  Sandbox: https://app.sandbox.midtrans.com/snap/snap.js
  Production: https://app.midtrans.com/snap/snap.js
* Gunakan data-client-key dari config('midtrans.client_key').

Button Bayar Sekarang:

* On click, fetch POST /bookings/{booking}/pay.
* Kirim CSRF token.
* Terima snap_token.
* Jalankan window.snap.pay(snap_token).

JS callback:

* onSuccess: reload atau redirect ke detail pesanan.
* onPending: reload checkout.
* onError: tampilkan alert gagal.
* onClose: tampilkan info bahwa pembayaran belum diselesaikan.

PENTING:
Jangan update database menjadi paid dari callback JS.
Database hanya boleh diupdate dari webhook Midtrans.
